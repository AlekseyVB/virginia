# Архитектура и поток данных

## Общий процесс

```text
Видео или изображения
        |
        +--> извлечение кадров (FFmpeg / notebook)
        |
        +--> ручная разметка в FORA
                  |
                  +--> FORA_ObjectsData_*.pickle
                              |
                              +--> FORA_to_YOLO_DataConverter.py
                                          |
                                          +--> YOLO *.txt

Видео + веса U-Net
        |
        +--> Frames_Predictions_Player_X4_grayscale.py
                  |
                  +--> маска класса «ковёр» и визуальный просмотр
```

## FORA

FORA — настольный разметчик на PySimpleGUI. Он открывает каталог изображений,
позволяет рисовать прямоугольники, хранит координаты и метаданные в Pandas и
сохраняет их в pickle-файл. Канонической в текущей структуре считается версия
`v1.1.1`; более ранние файлы оставлены для исторической совместимости.

Формат pickle удобен для продолжения работы в Python, но небезопасен для
загрузки из недоверенных источников. Для обмена данными лучше добавить экспорт
в JSON, CSV или один из стандартных форматов аннотаций.

## Конвертер YOLO

`FORA_to_YOLO_DataConverter.py` преобразует прямоугольник
`(x_min, y_min, x_max, y_max)` в нормализованные координаты YOLO:

```text
class x_center y_center width height
```

Номер класса сейчас задаётся константой `CLASS_NUM`. Для нескольких классов
его следует брать из метаданных объекта, а не из глобальной настройки.

## U-Net inference

`Frames_Predictions_Player_X4_grayscale.py` содержит архитектуру U-Net,
загружает внешние веса HDF5 и выполняет сегментацию разностей соседних кадров.
Выход модели преобразуется в двухцветную маску: фон и ковёр.

Код объединяет архитектуру сети, обработку видео и GUI в одном файле. При
дальнейшем развитии рекомендуется выделить:

- `model.py` — создание и загрузка U-Net;
- `video.py` — чтение и подготовка кадров;
- `ui.py` — интерфейс просмотра;
- `config.py` — пути, размеры и палитру классов.

## Внешние артефакты

Для полного запуска нужны данные, которых нет в Git:

- веса `Models_Weights/unet_carpets_weights_grayscale.h5`;
- входные видео;
- сохранённые файлы разметки FORA;
- при необходимости обучающий датасет.

Крупные веса лучше публиковать как GitHub Release или хранить через Git LFS с
описанием версии, контрольной суммы и условий использования.
